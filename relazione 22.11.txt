Nel diagramma si può osservare come vengono utilizzati due firewall per segmentare la rete interna e quella DMZ.
La DMZ è quindi una parte di rete o una sottorete all'interno della quale inseriamo determinati servizi/server pubblici (ex. server WEB ) che rischiano di essere attaccati ,così se venissero attaccati e quindi compromessi non potrebbero creare problemi all'Intranet (rete interna), dove si trovano server che si vogliono tenere al sicuro come ad esempio un  server database.
Lo scopo di una configurazione di questo tipo è far si  che i server all'interno della rete interna , di solito quelli dove troviamo dati importanti per un azienda possano essere protetti in modo migliore .
Nel firewall esterno possiamo settare le regole sia per il server web che per quello mail smtp. Settiamo quindi che le richieste dagli ip che hanno :ip source any , ip destination del web server ,impostata porta 80 , protocollo tcp come regola ,verranno ridiretti al web server. Analoga cosa per il server SMTP con la differenza  che in questo caso la porta è la  25.
Il firewall interno quindi isola la rete interna e non può essere quindi compromessa in caso che i server nella zona DMZ siano stati attaccati e infettati da virus. Questo metodo con 2 firewall viene detto metodo a cuscinetto. Il firewall interno quindi blocca il traffico in entrata, isolando la rete interna da minacce esterne.Si può anche creare una DMZ con un solo firewall, ma è un metodo meno sicuro , tuttavia settare un sistema con 2 firewall nonostante sia molto più sicuro, risulta più complicato da configurare e comporta costi più onerosi per una azienda.
